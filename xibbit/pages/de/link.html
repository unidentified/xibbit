Mit dieser Seite können Sie einen Link erstellen, der Server, zu betretende Räume und ein Theme automatisch einstellt. Wer auf einen solchen Link klickt, erhält eine Login-Form, oder falls es sich um einen Link zum unregistrierten Betreten des Raumes handelt, ein Feld zur Eingabe eines Nicknames.
Bitte beachten Sie, dass bei Links, die nicht zum unregistrierten Chatten gedacht sind, User bereits einen XMPP Account benötigen um sich einloggen zu können. Üblicherweise sollten sich die User beim verwendeten Server registrieren.
Unregistriertes Chatten wird nur von einem Teil von Servern unterstützt. Falls Sie einen Link zum unregistrierten Chatten in einem Raum auf einem solchen Server erstellen, erstellen Sie möglichst auch noch einen für registrierte User. Geben Sie Usern auch die Webseite des verwendeten Servers, sodass diese sich registrieren können.
Falls Sie einen Link zum unregistrierten Chatten erstellen, geben Sie bitte nur registrierte (persistente) Räume an: Unregistrierte User können keine unregistrierten Räume betreten, sofern sich in diesen nicht bereits registrierte Besucher befinden.
Die Loginform für registrierte User beim Klicken auf einen Chatlink möchte üblicherweise die komplette JID ("user@server") als Usernamen. Bitte teilen Sie dies den Usern mit.

    ... oder geben Sie im Feld unten eine URL zu einem Style ein (eine funktionierende URL zu einer CSS-Datei, einem Candy Chat style). Lassen Sie das Feld leer falls Sie einen Style aus der Liste oben auswählen.
    Falls Sie einen Style schreiben möchten, verwenden Sie light.css als Beispiel. Firefox und Chrome sowie einige andere Browser erlauben das Editieren von CSS in der Browserkonsole, sodass das Erstellen von Styles relativ leicht ist (natürlich müssen Sie die Änderungen speichern).

  • ALLE Nachrichten in diesem Fenster, Gruppen- und Privatchats, werden mit dem Verschlüsselungspasswort ent- und verschlüsselt. D.h., alle Teilnehmer müssen das selbe Passwort verwenden, um kommunizieren zu können, falls ein Verschlüsselungspasswort verwendet wird. Nicht oder mit anderem Passwort verschlüsselte Nachrichten werden gar nicht erst angezeigt.
    Es "muss" kein Verschlüsselungspasswort genutzt werden, aber es kann manchmal die Privatssphäre deutlich erhöhen: Nur wer das Passwort kennt, kann die Nachrichten lesen, noch nicht einmal der Server (dem das Passwort nie mitgeteilt wird).
    Die Verwendung eines solchen Passworts setzt allerdings voraus, dass allen Teilnehmern das Passwort bzw. der das Passwort enthaltende Link bekannt ist, und dass Passwort bzw. Link sicher ausgetauscht werden können. Um die Verschlüsselung zu verwende, müssen zudem alle Nutzer einen Client einsetzen, der diese Art der Verschlüsselung unterstützt - aktuell ist das nur diese Webseite. Das Verschlüsselungspasswort der NIE zugleich Raum- oder Loginpasswort sein, da es dem Server nicht bekannt sein darf!
