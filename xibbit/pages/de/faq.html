Was ist XMPP?

XMPP ist ein Protokoll für Sofortnachrichten (Instant Messaging, IM) und Gruppenchats (MUC - Multi User Chat). Es definiert ähnliche Funktionen wie Sie sie vielleicht von ICQ, Google Talk (XMPP-basiert) oder Skype kennen.
"Protokoll" bedeutet hier, dass XMPP ein technischer Standard ist, der festlegt, wie XMPP-Server und XMPP-Clienten sich verhalten, d.h. z.B. welche Funktionen es gibt, welche Funktionen es noch zusätzlich geben kann, wie Nachrichten versendet werden usw.
Dabei werden die Begriffe XMPP und Jabber synonym verwendet - die Umbenennung des Jabber-Standards zu XMPP hatte rechtliche Gründe.

Es gibt mittlerweile relativ viele XMPP Server, und ebenfalls viele davon erlauben kostenloses Registrieren und Nutzen. Ein Großteil davon unterstützt Gruppenchat.
Üblicherweise registriert man sich auf der Homepage des XMPP Servers, nur ein paar wenige erlauben oder verlangen das Registrieren über einen Clienten.

XMPP Clienten sind entweder Multimessenger wie z.B. Pidgin, oder dedizierte XMPP Clienten wie z.B. Psi oder einige Smartphone-Apps. Diese "Auflistung" ist natürlich nicht vollständig, da es sehr viele XMPP Clienten gibt.
Neben installierten XMPP Clienten gibt es auch XMPP Clienten, die direkt im Browser laufen und dort ausgeführt werden. Diese können z.B. auf einer Webseite wie dieser sein, lokal gespeichert werden oder als Addon installiert. Dazu muss der Server Webzugriff (das sogenannte HTTP Binding) unterstützen - dies ist jedoch häufig der Fall, sodass es möglich ist, XMPP ohne installierten Clienten zu verwenden. Das ermöglicht auch eine reibungslose Nutzung z.B. mit dem Tor Browser oder auf Smartphones.
XMPP Clienten im Browser sind z.B. diese Webseite, converseJS, iJab oder Jappix und Jappix Mini.

User auf einem XMPP Server werden durch eine JID (Jabber ID) identifieziert. Eine JID ist für XMPP/Jabber, was eine Mailadresse für Mail ist. Der User kann sich mit seiner JID und seinem Passwort, die in der Form username@server ist, einloggen (falls ein Einloggen in einem Client mit dem Usernamen als Usernamen nicht funktioniert, immer die gesamte JID als Username probieren). Andere User können den User unter dieser JID anschreiben.
XMPP Gruppenchat Räume haben ebenfalls eine JID, in der Form raumname@gruppenchatserver. Die URL des Gruppenchatserver ist üblicherweise etwas anders als die Server-URL, z.B. oft conference.server oder muc.server, z.b. bei jappix.com muc.jappix.com.
User in einem Gruppenchatraum lassen sich über ihren Nicknamen anschreiben, Moderatoren können auch die echte JID sehen (in manchen Räumen kann jeder User die JID der anderen sehen).

Wie funktioniert diese Webseite?

Diese Webseite funktioniert als XMPP Client für Gruppenchats. Sie funktioniert mit jedem XMPP Server, der XMPP MUC (Multi User Chat) und Webzugriff unterstützt.

Diese Seite ermöglicht Ihnen, sich einzuloggen und Gruppenchats zu betreten. Nutzen Sie dafür die Login-Seite. (Bitte beachten: Die Formulare möchten üblicherweise eine JID als Username.)
Diese Seite erlaut es zudem, Links zum automatischen Betreten von Räumen zu erstellen. Diese speichern Server und Raumname(n) schon im Link, sodass ein User, der darauf klickt, nur noch eine Loginform (JID und Passwort) ausfüllen muss, oder nur einen Nicknamen eingeben, falls es sich um einen Link für unregistriertes Betreten handelt (unregistriertes Betreten von Gruppenchats wird nicht von allen Servern unterstützt).

XMPP Gruppenchats können auch über diese Webseite registriert und administriert werden. Bitte lesen Sie dazu die Infoseite.

Wie funktioniert XMPP MUC (Multi User Chat)?

XMPP Multi User Chat Räume werden erstellt sobald ein (registrierter) User versucht, einen Raum zu betreten, der noch nicht existiert. Ein solcher Raum existiert solange, wie sich Besucher in ihm befinden. Der User, der den Raum zuerst betreten hat, ist Raumeigentümer.
Der Raumeigentümer kann den Raum registrieren, indem er ihn konfiguriert und als persistent einstellt. Tut er dies nicht, so verschwindet der Raum und alle gemachten anderen Einstellungen wieder, sobald alle User den Raum verlassen haben.

User in einem XMPP MUC Raum haben eine sowohl eine Zugehörigkeit (affiliation) als auch eine Rolle (role).
Eine Zugehörigkeit ist persistent (permantent). Hier gibt es "owner" (Raumeigentümer - diese haben die meisten Rechte und können den Raum konfigurieren, es kann mehrere geben), "admin" (Admins), "member" (als Raummitglieder eingetragene Nutzer) und "outcast" (gebannte User). Der Server speichert die Zugehörigkeit basierend auf der JID, sodass ein User diese auch behält, wenn er den Raum verlässt und unter dem selben oder einem anderen Nicknamen wieder betritt. Die Zugehörigkeit bleibt solange erhalten wie der Raum existiert oder sie manuell verändert wird. Normale aktuell anwesende User ohne administrative Privilegien oder Mitgliedsstatus haben die Zugehörigkeit "none", also keine bestimmte.
Eine Rolle dagegen ist nicht persistent. Sie bleibt so lange erhalten wie der User sich im Raum befindet. Normale User haben die Rolle eines Teilnehmers ("participant"), Admins und Eigentümer sowie temporäre Moderatoren die Rolle eines Moderators ("moderator"). Temporäre Moderatoren sind normale User, deren Rolle von einem Eigentümer oder Admin auf Moderator ("moderator") gesetzt wurde. Ihr Moderatorstatus verschwindet wieder sobald sie den Raum verlassen. Kicken funktioniert, indem ein Moderator die Rolle des zu kickenden Users auf "none", also keine mehr, setzt - damit verlässt der User automatisch den Raum.

Admins können kicken, bannen, und Usern permanenten Mitgliedsstatus geben. Raumeigentümer können das selbe wie Admins, aber zusätzlich v.A. den Raum konfigurieren (z.B. ihn registrieren oder löschen, die maximale Anzahl der Nutzer setzen, ein Raumpasswort setzen oder entfernen, nur Mitglieder als Nutzer erlauben etc.). Temporäre Moderatoren können vorrangig User kicken, und ein paar kleinere Dinge wie z.B. in manchen Räumen das Thema setzen in denen normale User dies nicht dürfen.

Normalerweise kann ein Raum von allen Usern auf dem Server betreten werden, manchmal auch von Usern von anderen Servern. Allerdings können Raumeigentümer den Zugriff zu einem Raum einschränken. Räume können auch nur für Mitglieder zugänglich (members only) oder durch ein Zugangspasswort geschützt werden. Bei einem Raum, den nur Mitglieder betreten können, muss ein Admin neuen Usern, die vorher noch kein Mitglied waren, Mitgliedsstatus geben.

Auf der Infoseite finden Sie mehr Informationen über XMPP und Raumadministration.

Welche Befehle funktionieren im Chat?

Der XMPP-Client auf dieser Webseite unterstützt einige einfache Befehle, die z.B. zum Betreten und Verlassen von Räumen, dem Öffnen von Privatchats, dem Auflisten aller Nutzer im Raum oder dem Ignorieren und oder dem Aufheben des Ignorierens von Usern dienen.
Er unterstützt auch administrative Befehle, d.h. die Verwaltung eines Raums über diese Seite.
