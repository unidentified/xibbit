Einfache Befehle

Bitte beachten: Verwenden Sie die FAQ für weitere Informationen zu XMPP und zur Verwendung dieser Seite.

/join [room]
Beispiel 1: /join raum1 (Raum raum1 auf dem aktuell verwendeten Chatserver betreten).
Example 2: /join raum1@conference.someserver.com (Raum raum1 auf dem Gruppenchat-Server conference.someserver.com betreten - bitte beachten Sie, dass nicht alle XMPP Server das Betreten von Räumen auf anderen Servern unterstützen, daher kann es sein, dass Sie einen Raum auf einem anderen Server als dem aktuell verwendeten nicht betreten können).
Example 3: /join raum1 meinraumpasswort (Den passwortgeschützten Raum raum1 betreten, wenn das Raumpasswort "meinraumpasswort" lautet.)

/part
Den aktuellen Raum verlassen. (Bitte beachten: Privatchats müssen aktuell grafisch geschlossen werden, indem der Tab geschlossen wird - klicken Sie auf das "x").

/names
Auflisten aller User im aktuellen Raum. Nutzen Sie diesen Befehl auf kleinen Screens (z.B. Smartphone), da dort keine Userliste angezeigt wird.

/query [nick]
Ein Privatchatfenster für einen Privatchat mit dem User unter dem angegebenen Nick öffnen.
Beispiel: /query user1 (startet einen Privatchat mit dem User user1).

/ignore [nick]
Ignorieren des Users unter dem angegebenen Nicknamen.
Beispiel: /ignore user1 (ignoriert den User user1 in diesem Raum).

/unignore [nick]
Nicht mehr ignorieren des Users unterm angegebenen Nicknamen.
Beispiel: /unignore user1 (den User user1 nicht mehr ignorieren).

/themes
Alle verfügbaren Themes auflisten.

/theme [theme]
Beispiel: /theme light
Ein Theme Laden. Listen Sie verfügbare Themes über /themes auf.

Administrative Befehle

Bitte beachten: Verwenden Sie die Info-Seite zu XMPP und Raumadministration für weitere Informationen zu XMPP und zur Verwaltung eines Raumes.

/config
Konfigurieren des aktuellen Raums (öffnet ein Konfigurationsformular).

/set {affiliation | role} [wert] [user]
Setzt die Zugehörigkeit (affiliation) oder Rolle (role) eines Users auf einen bestimmten Wert. "user" kann der Nickname des Users sein falls dieser sich gerade im Raum befindet, oder seine JID.
Beispiel 1: /set affiliation member test1@myserver.com (setzt den User mit der JID test@myserver.com als permanentes Mitglied (member) des aktuellen Raums).
Beispiel 2: /set affiliation admin user1 (setzt den User mit dem Nicknamen user1 im aktuellen Raum als Administrator - der Server speichert die Zugehörigkeit automatisch und erkennt sie auch wieder, wenn der User den Raum verlässt und wieder betritt).
Beispiel 3: /set role moderator user2 (gibt dem User mit dem Nicknamen user2 die Rolle eines temporären Moderators, i.e. der User bleibt so lange Moderator wie er im Raum ist).

/list [owner | admin | member | none | outcast | moderator | participant]
Auflisten aller User mit einer bestimmten Zugehörigkeit (affiliation) oder Rolle (role). Beachten Sie, dass Zugehörigkeiten auch erhalten bleiben, wenn der User nicht im Raum ist (außer "none", d.h. keine bestimmte Zugehörigkeit - dies trifft für normale User ohne member-Zugehörigkeit zu) - die ausgegebene Liste zeigt hier die JID an, nicht den Nicknamen.
Beispiel 1: /list admin (liste alle User mit der Zugehörigkeit "admin", also alle Admins des Raumes, auf).
Beispiel 2: /list outcast (liste alle User mit der Zugehörigkeit "outcast", also alle gebannten User, auf).
Beispiel 3: /list moderator (liste alle User mit Moderatorrolle auf, das sind neben temporären Moderatoren auch anwesende Raumeigentümer und Admins).
Beispiel 4: /list none (liste alle User ohne spezielle Zugehörigkeit auf - das sind die anwesenden normalen User ohne member-Status).

/ban {user}
Bannt einen User (für "user" kann der Nickname eingesetzt werden falls der User anwesend ist, oder generell die JID).

/unban {jid}
Einen User nicht mehr bannen (JID des Users einsetzen).
Bitte beachten (!): In Räumen, die nur von Mitgliedern betreten werden können, muss statt oder zusätzlich zu /unban der Befehl /set affiliation member {jid} verwendet werden, um dem User wieder Mitgliedsstatus zu geben.

/kick {user}
Einen User aus dem Raum kicken (Nickname angeben, nicht die JID).
