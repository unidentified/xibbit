If you want you and other users to be able to join your room(s) by just clicking a link, you can fill out the form below and create a join link.
Please note that users usually need an account at the server you are using to join, or at least a XMPP account at another server if the server you've chosen supports users from different servers. So if most of your users don't have a XMPP account yet (or none at this server), you should give them a link to the servers homepage so they can register.
When you allow non-registered users to connect, please only specify registered (persistent) rooms for joining: They cannot join non-persistent rooms on most servers. Not all servers allow anonymous (non-registered) connections, so choose one that does.
If users can connect unregistered, they will just have to enter a nickname to chat. If they connect registered, they will see a login form and have to enter their JID/ username ("user@server") and their password to chat. If you are using a server that allows non-registered users to chat, you could create one link that allows unregistered connection (link for guests) and another link that doesn't do so (for registered users).

  • Select a style from the list below ...

    ... or enter an URL (valid URL to a CSS-file, a Candy-Chat style) below (if you enter anything below, this URL will be used, no matter what is selected above). Leave the field empty to use one of the styles on the list. If you want to write a style, use light.css as an example. The browser console in Firefox and Chrome allows you to inspect, change and create styles in browser easily.

  • If you allow non-registered users to connect, please only specify registered rooms for joining: They cannot join non-persistent rooms on most servers.

  • If you specify an encryption password, all messages in the window opened using the link will be automatically encrypted using the password.
    As the password is part of the link you'll receive below, all messages (groupchat and private chats) will only be readable for those who have the link.
    This has advantages concerning security (only who has the password/ link will be able to read the messages, not even the server, as the password is never sent anywhere), but disadvantages concerning compatibility (users with other XMPP clients will not be able to participate in chat as they cannot decrypt the messages they receive). So only use this if you really need it. To use no password, just leave the field below empty.
    Make sure the encryption password is never the same as a room password or a login password.
    Please not that nicknames, room names and subjects, commands etc. are not encrypted - only the actual messages are.
